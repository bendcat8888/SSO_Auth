Developer Demo

See how another web app should launch and receive Innogen SSO.

This page is the live reference for partners integrating with https://sso.innogen-pharma.com. It shows the exact redirect pattern, callback check, and the button treatment you can reuse in your own product.

Live entry button

When the user clicks Continue with Innogen, send them to https://sso.innogen-pharma.com/login?return_to=... where return_to is your app's callback page, not your homepage.

What this demonstrates

Use your callback page in return_to. After the SSO flow completes, that callback page should call /api/v1/verify-session with credentials: "include".

Integration flow

These are the only three moving parts the consuming app needs to understand.

1

Start on your app

Render a "Continue with Innogen" button and redirect to the hosted login page with your callback page URL.

2

User signs in at SSO

The hosted page detects the method, handles Google or app password, then sets the SSO cookie.

3

Return and verify

Your callback page calls the SSO verify endpoint, reads the authenticated user, and continues your app flow.

Snippet: launch the hosted login

Embed this in the consuming application where the user begins sign-in. The return_to value should be your callback page.

<button id="continueWithInnogen" type="button">
  <img src="https://sso.innogen-pharma.com/static/favicon.png" alt="Innogen" />
  Continue with Innogen
</button>

<script>
  const SSO_BASE = "https://sso.innogen-pharma.com";
  const returnTo = window.location.origin + "/sso-callback.html"; // your callback page

  document.getElementById("continueWithInnogen").addEventListener("click", () => {
    window.location.href = `${SSO_BASE}/login?return_to=${encodeURIComponent(returnTo)}`;
  });
</script>

If you pass your homepage instead of a callback page, that page itself must still run the session verification logic. The usual setup is a dedicated callback route such as /sso-callback.html.

Snippet: callback verification

Your callback page should verify the Innogen cookie and then create or refresh your own app session.

Open live callback page Download sso-callback.html 
<script>
  const SSO_BASE = "https://sso.innogen-pharma.com";

  fetch(`${SSO_BASE}/api/v1/verify-session`, {
    method: "GET",
    credentials: "include"
  })
    .then((response) => response.json())
    .then((data) => {
      if (!data.authenticated) {
        window.location.href = "/login.html";
        return;
      }

      console.log("SSO user:", data.user);
      // Create or refresh your own app session here.
    });
</script>
{
  "authenticated": true,
  "user": {
    "email": "name@innogen-pharma.com",
    "domain": "innogen-pharma.com"
  }
}

Your callback URL must be an allowed host under innogen-pharma.com, innogen-pharma.ph, or innogen.local. Local development can use HTTP only for *.innogen.local.
