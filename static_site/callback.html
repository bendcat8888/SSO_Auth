Waiting for /api/v1/verify-session...
This page is only the callback reference. In a real app, after /api/v1/verify-session returns authenticated: true, your callback page should send data.user to your own backend so your app can create its own local session. Also replace the example redirect target with your app's real default page. 
fetch("/api/session/sso-login", {
  method: "POST",
  headers: { "content-type": "application/json" },
  credentials: "include",
  body: JSON.stringify({ user: data.user })
})
  .then((response) => response.json())
  .then(() => {
    window.location.href = "/";
  });
